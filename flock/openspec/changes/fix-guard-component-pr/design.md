## Context

PR #382 adds `GuardComponent` (abstract framework) and `AzurePromptShieldGuard` (Azure Content Safety implementation) to Flock's agent component system. The code was generated by GitHub Copilot's SWE agent and is architecturally sound — 40 tests pass, the abstraction fits cleanly into `AgentComponent` lifecycle hooks. However, four issues block merge: a CodeQL failure, a missing `SecretStr` pattern, a wrong package manager reference, and test assertions that trigger static analysis.

**Current state of the files:**
- `src/flock/components/agent/azure_prompt_shield.py` — 221 lines, working implementation
- `src/flock/components/agent/guard.py` — 246 lines, clean abstract framework (no changes needed)
- `tests/test_guard_component.py` — 603 lines, 40 tests all passing

## Goals / Non-Goals

**Goals:**
- Make CI green (CodeQL + Backend Quality)
- Align with existing Flock patterns (`SecretStr`, `uv`)
- Keep changes minimal — don't refactor what Copilot got right

**Non-Goals:**
- Adding new features to the guard framework
- Changing the guard architecture or API surface
- Adding env var resolution pattern for api_key (follow-up)
- Endpoint URL validation / SSRF protection (follow-up)

## Decisions

**1. SecretStr for api_key**

Change `api_key: str` → `api_key: SecretStr | None` on `AzurePromptShieldConfig`. This matches the pattern established in `openclaw/config.py:47` where `token: SecretStr | None` is used for the gateway token. The `_build_headers` method needs to call `.get_secret_value()` and the empty-string fallback becomes a None check.

Alternative considered: Leave as `str` and file a follow-up. Rejected because commit `862ca232` literally just fixed this exact pattern — shipping another plaintext secret field would be inconsistent.

**2. CodeQL URL assertion fix**

The test at line 508 asserts `"env.cognitiveservices.azure.com" in call_url`. CodeQL flags this as "incomplete URL substring sanitization" — the substring could appear at any position. Fix by asserting the full expected URL: `assert call_url == "https://env.cognitiveservices.azure.com/contentsafety/text:shieldPrompt"`.

Alternative considered: `call_url.startswith(...)`. Rejected because asserting the exact URL is more precise and equally simple.

**3. Error message: pip → uv**

Line 208: `"Install it with: pip install azure-identity"` → `"Install it with: uv sync --extra azure"`. Flock uses `uv` exclusively — the AGENTS.md, README, and all other error messages reference `uv`.

## Risks / Trade-offs

- **[Low] Test breakage from SecretStr** — Tests construct `AzurePromptShieldConfig(api_key="test-key-123")`. With `SecretStr`, this becomes `api_key=SecretStr("test-key-123")`. The header assertion `headers["Ocp-Apim-Subscription-Key"] == "test-key-123"` stays the same since we test the resolved value. → Mitigation: run tests after each change.
- **[Low] CI may not trigger on push** — The Copilot agent branch may need a human push to trigger Quality Gates. → Mitigation: push the fix commit, verify CI runs.
